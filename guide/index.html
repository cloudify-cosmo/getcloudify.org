Cloudify User's Guide

This guide is written for application developers and administrators. The guide provides all the information and tools needed to install, setup and leverage Cloudify for on boarding your applications to any cloud environment. It includes an overview section, getting started guide, reference documentation, APIs, configurations and other topics.

Note that this guide is specific to Cloudify developers.

  • Getting Started

    This section contains a short Quick Start Guide to get you up and running with Cloudify in minutes.

    • Quick Start Guide - 10-Minute Quick Start Guide

    Product Overview

    This section contains a comprehensive overview of how Cloudify works, how it's architected and what are its main components.

    • Introduction to Cloudify - Architecture overview, design principles and key features of Cloudify
    • The Universal Service Manager - Describes the Universal Service Manager (USM), a key components that allows Cloudify to run and monitor any kind of middleware stack.

    This section contains information about how to install Cloudify and configure for usage on various clouds.

    • Command Line Reference - Lists the available interactive shell commands and how to use them.
    • Configuring for Public Clouds - Explains how to Configure Cloudify to run your application on public clouds.
    • Installing on a Private Cloud - Explains how to setup Cloudify to run your application on a private cloud.
    • Cloudify on Windows Azure - Explains how to install Cloudify on Windows Azure

    Developing Recipes

    This section explains the concept and practicalities of application recipes, which the mechanism that allows Cloudify to support any kind of middelware stack

    • Anatomy of a Recipe - An overview of the recipe framework, terminology and syntax.
    • The Application Descriptor - Describes the part of the recipe that defines which application tiers compose your application and their inter-dependencies.
    • The Service Descriptor - Describes the part of the recipe that defines a specific application tier (or service), how to setup them up and have Cloudify manage them.
    • Lifecycle Events - Describes the order of events in Cloudify application and service lifecycle.
    • More...
    • Recipe Parameters - Describes how to parameterize a recipe and externalize key configuration elements
    • Extending Recipes - Explains how to create new recipe from existing one by overriding only the elements you wish modify.
    • The Service Context API - Describes the Service Context API, which is Cloudify's API for interacting with an applications service's runtime environment and context.
    • Attributes API - Describes the Attributes API, which is used for sharing information between services of the same application
    • Custom Commands - Describes the custom commands mechanism and usage
    • SLA Requirements - Explains how to specify SLA elements per application service, e.g. initial number of instances.
    • Monitoring Probes - Describes the use of monitoring probes and plugins to monitor service status, performance and availability
    • Configuring the Web Based UI - Explains how to configure the web based UI to present the information generated by probes and monitoring plugins.
